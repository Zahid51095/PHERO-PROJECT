1.Discuss the scope of var, let, and const

var: Scoped to the function or global context. Gets hoisted (moved to the top) within its scope, so it can be accessed before it's declared. Can lead to unexpected behaviors due to hoisting and wider scope.

let: Scoped to the block (curly braces) it's in, like an if statement or loop. Doesn't get hoisted, so it can only be accessed after it's declared. Safer and more predictable for most cases, especially in modern JavaScript.

const: Also scoped to the block it's in, just like let. Like let, doesn't get hoisted and can only be accessed after it's declared. Must be assigned a value when declared and cannot be reassigned afterwards. Ideal for values that won't change, making your code more readable and less error-prone.

In short, prefer using let and const due to their clearer scoping rules and avoidance of common pitfalls associated with var. Use const when you want to keep a value constant, and use let for variables that can change their value.

2.Tell us the use cases of null and undefined

Undefined- Indicates an uninitialized variable, an absent value, or the default return value of a function with no explicit return.

Use cases: Declared variables without values, missing function arguments, checking property existence.

Null: Explicitly represents an empty or no-value state, often set by the programmer.

Use cases: Assigning no value to a variable, indicating absence in objects or APIs, representing emptiness intentionally.

3.What do you mean by REST API?

A REST API (Representational State Transfer Application Programming Interface) is a standardized way for software applications to communicate over the internet. It follows a set of rules and conventions that allow clients and servers to exchange data using standard HTTP methods like GET, POST, PUT, and DELETE. It's commonly used in web development to create web services and enable different applications to interact with each other.
